NextDeploy Project

NextDeploy is a deployment system of virtualized development environments in a compute cloud.

Generic installation templates are defined for different frameworks or cms (currently symfony, drupal, wordpress). When creating a project, a git repository is associated with this installation template. Thus, virtual machines can be run on demand by deploying the site on a preinstalled system following prerequisites specified in the template.

Below, the Big picture, Features and User-Interfaces (Web, CLI, and Android) of the project. On Big picture, there are the main node at left, compute cloud (openstack) at center and monitoring node at right.


Openstack Cloud

The project uses a cloud to host vms. Curently, NextDeploy is working with openstack.

Puppet Patterns

The users ask to test and install their web application on a vm. These installation templates are defined with the puppet tool.

Git Repository

Users need a code repository to store their work. Git is used for versioning of project developments. Gitlab manager is installed for these needs.

REST api

A REST API (in ruby on rails) is the intermediary between these 3 last systems (Cloud, Puppet, Git) and can manage user authentication, project creation, adding users, and of course the launch of vms.

Local Installation

A "magical" script manages the complete installation* (only sudo and curl are prerequisites) on local computer (OSX, Debian, Ubuntu or Fedora are supported). But it needs a lot of ram (>=12Go) and a powerful cpu. The installation is done by vagrants tools with 6 defined virtual machines (1 for core node, 4 for openstack nodes and 1 for monitoring node). Once completed, you can start webui in your browser and start working with projects and VMS. But the vms are launched inside a compute node, .. which is already a vagrant vm. So this causes a very long (~1h30 vs 20min on real cloud) VM creation process.

* This script installs some needed tools for execute NextDeploy on local computer: virtualbox or kvm, git, and vagrant. And also, the script changes both files /etc/hosts and /etc/resolv.conf (add -r parameter to disable that). He must be used at your own risk. 

[erifeh@pm-erifeh nextdeploy]$ ./scripts/./setup -h
Usage: ./scripts/./setup [options]

For example:
./scripts/./setup -y

Remote Installation

An other "magical" script makes remote installation easier and manages lot of tasks for you. 

Usage: ./scripts/./setup-remote [options]

For example (change fqdn and domain with yours), adding a third nova node:
./scripts/./setup-remote --puppetmaster-ip XXX.XXX.XXX.XXX --puppetmaster-fqdn puppetnextdeploy --domain nextdeploy.io -y --nova3-ip XXX.XXX.XXX.XXX
